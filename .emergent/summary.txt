<original_problem_statement>
The user wants to create a social media mobile app called BEAT MATES for dancers.

**PRODUCT REQUIREMENTS:**
- **User System:** Users can register and log in using both Google social login and traditional email/password.
- **Dance Disciplines:** Upon registration or from their profile, users can select multiple dance disciplines they are interested in.
- **Social Feed:** The main home feed should display content (videos, photos, text) from other dancers who are part of the same selected dance disciplines. The feed should look and feel like Instagram.
- **Paid Lessons:** Any user can offer paid dance lessons.
  - **Pre-recorded Lessons:** Users can upload and sell pre-recorded video lessons.
  - **Live Lessons:** Users can make themselves available for live, one-on-one video lessons.
- **Live Lesson Flow:**
  1. A student finds an available professional/teacher on a dedicated page.
  2. The student sends a request for a 30-minute live lesson.
  3. The student must pay for the lesson in advance.
  4. If the professional accepts the request, a paid video call starts.
- **Availability System:** Users have a toggle in their profile to set their real-time availability. They can also use a calendar to schedule future availability, allowing other users to book lessons in advance.
- **Payment Systems:** The app should support standard payment methods like Stripe and PayPal. The user does not have API keys yet and requires a demo/mocked version first.
- **Video Call Integration:** The user has no preference for the video call provider, as long as it works live.
- **Design:** The app must strictly follow the UI/UX provided in the Figma designs. This includes a dark theme, a specific coral accent color (), and the BEAT MATES logo.
- **User Testing:** The user wants to be able to have 5 real friends register on the app to test the core functionalities like registration, category selection, posting, and seeing each other on the available teachers page.

</original_problem_statement>
